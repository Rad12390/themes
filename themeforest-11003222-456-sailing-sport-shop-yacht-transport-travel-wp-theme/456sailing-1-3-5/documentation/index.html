Theme Compatibility

Requirements

Many issues you may run into such as demo content fails when importing, plugins conflict in admin panel all these issues are related to low PHP configuration. The solution is to increase the PHP limits.

PHP Configuration Limits
  • max_execution_time 300
  • memory_limit 128M
  • post_max_size 32M
  • upload_max_filesize 32M
  • max_input_vars 3000
How To Check PHP Limits

Check your current PHP limit by installing Wordpress phpinfo() plugin.

How To Increase PHP Limits

Follow these 5 tips to increase your web-host’s PHP limit.

Installation

Wordpress Installation

  • Navigate to "Appearance" => "Themes" => "Add New", and click the "Upload Theme" button.
  • Navigate to find the zipped file on your computer and hit "Install Now"
  • Activate the installed theme by going to "Appearance" => "Themes" and clicking the "Activate" button.
  • Once you activate the theme, you will see the notification message, click the "Begin installing plugins" button to go to "Install Required Plugins" where you need to install and activate the required plugins.
  • Click the "Install" button on each plugin to start installing them and activate them after installation.

Uploading Issue
  • If you get the "Are You Sure You Want To Do This" message it means your PHP "upload_max_filesize" upload file size limit doesn't allow you to do this. If it happens install/update the theme via FTP or contact your hosting company and ask them to increase your "upload_max_filesize" limit.

FTP Installation

  • Log into your hosting via an FTP client.
  • Unzip theme zipped file and only use the extracted theme folder.
  • Upload the extracted theme folder into "wp-content" => "themes" folder.
  • Activate the installed theme by going to "Appearance" => "Themes" and clicking the "Activate" button.
  • Once you activate the theme, you will see the notification message, click the "Begin installing plugins" button to go to "Install Required Plugins" where you need to install and activate the required plugins.
  • Click the "Install" button on each plugin to start installing them and activate them after installation.

Theme Update

Auto Updater

The easiest way to update your theme is to use Auto Updater.

  1. Auto Updater can only be used after purchase code verification.
  2. Navigate to "Appearance" => "Theme Options" => "General Settings".
  3. Find "Purchase Code" field and enter your purchase code for verification. Find Your Purchase Code.
  4. And click to "Save Changes".
  5. Once you have verify the auto updater, you will get notified in the "Dashboard" => "Updates" and "Themes" section when a new theme update is ready to be installed.
Traditional Method
  • Log into your "ThemeForest" account and navigate to your "Downloads" and find the theme purchase.
  • Find "Download" button and choose to download the 2.)"Installable WordPress file", which is installable Wordpress theme (zipped) file.
  • Decide how you want to install the update via FTP or via WordPress

  • Upload via FTP

    • Go to "wp-content" => "themes" folder, find theme folder and delete it, your content and theme settings will not be lost.
    • Unzip 2.) installable Wordpress theme (zipped) file and only use the extracted theme folder.
    • Upload the extracted theme folder into "wp-content" => "themes" folder.
    • Activate the installed theme by going to "Appearance" => "Themes" and clicking the "Activate" button.
    • If theme update includes plugin updates you will see notification message.

  • Upload via Wordpress

    • Deactivate current theme version in "Appearance" => "Themes" by activating a different theme, once the theme is deactivated delete it, your content and theme settings will not be lost.
    • Navigate to "Appearance" => "Themes" => "Add New", and click the "Upload Theme" button.
    • Navigate to find 2.) installable Wordpress theme (zipped) file on your computer and hit "Install Now"
    • Activate the installed theme by going to "Appearance" => "Themes" and clicking the "Activate" button.
    • If theme update includes plugin updates you will see notification message.

  • Uploading Issue

    • If you get the "Are You Sure You Want To Do This" message it means your PHP "upload_max_filesize" upload file size limit doesn't allow you to do this. If it happens install/update the theme via FTP or contact your hosting company and ask them to increase your "upload_max_filesize" limit.
    • If you get the "Destination folder already exists" after failed theme installation it means you have already the theme with the same folder name. We recommend you to use auto updater or you update the theme via FTP.

Importing Issues
  • If you get the "Sorry but your import failed" message or "500 Internal Server Error" error usually it means importing process doesn't work with your web-host. You have to contact your web-host and ask to increase your PHP "max_execution_time 300" and "memory_limit 128M" limits.
  • Portfolio pages are empty - your portfolio pages are empty because "Essential Grid" plugin doesn't have any published grids, import the content for "Essential Grid" plugin.
  • Revolution Slider Error: Slider with alias "slider-name" not found - you need to import slides for "Revolution Slider" if you see an error like that after importing process.
  • Main navigation doesn't look organized - XML dummy data file doesn't have the settings for "Menus", go to "Appearance" => "Menus" => "Manage Locations" to activate it in "Primary Menu" theme location.
  • Menu doesn't look like mega dropdown menu - it happens because standard Wordpress importer does not have these options to import custom menu settings. See this article how to set mega dropdown menu.
Importing Tips
  • If the importing process has been interrupted or you want to repeat it since the beginning, we recommend you to remove all content from your database. There is fast and easy way to get rid of content with Wordpress Reset plugin. "Wordpress Reset" plugin will remove all content from your database, it doesn't remove plugins, but will deactivate them and remove all pages, posts, menus, plugins data, widget data, theme settings and theme options.
  • If you need shortcode of one page please use our "Test Drive" demo version, you can copy the code of any page you need, see this article how to get access to "Test Drive" demo version.

The theme has OptionTree theme options panel that is loaded with options. Because of the large amount of options theme provides, we can't go over all of them in full details in our documentation. Please go to "Appearance" => "Theme Options" to take some time and navigate through each section to see what theme options offers you.

Translation Files

There are two ways to store translation files. The first method is storing it inside ..wp-content/themes/{theme-name}/languages/ folder, and the second method is storing it inside ../wp-content/languages/themes folder.

  1. Method (1)

    The location of "languages" folder in ../wp-content/themes/{theme-name}/languages/ folder, despite this method in the most popular, it has one downside. Whenever user updates the theme, the theme folder is replaced with a new updated theme folder and user loses all translated files.

    If you choose this method remember that the file structure is only the language name, for German language the file names would be de_DE.mo and de_DE.po.


  2. Method (2)

    Place your translation files in ../wp-content/languages/themes/ folder, create "language" and "themes" folder inside it.

    If you choose to use second method, remember that the file structure is different, translation files must include the theme name, for example, if the language files are in German language, the file names would be {theme-name}-de_DE.mo and {theme-name}-de_DE.po.

    IMPORTANT NOTE - Updating the theme will effect on wp-content folder. Storing translation files by this way is more safety.

How To Edit .po File

  1. Download POEdit software.
  2. Open default.po file in POEdit.
  3. Select the line you want to translate. In the lower section of the POEdit window, you can add the translation.
  4. When you translate all your needed text strings, save the file with language code. For German language the file names would be de_DE.po.
